# 0.1.4 — 2026-06-19

## Changed

- **`medcode_map_codes` `parents`/`children` documented as depth-1** ([#7](https://github.com/cyanheads/medical-codes-mcp-server/issues/7)). The tool description and `direction` field now state they return the immediate parent or children only — one level per call — and that a full ancestor/descendant path requires iterative calls.
- **`medcode_browse_hierarchy` ICD-10-PCS description corrected** ([#4](https://github.com/cyanheads/medical-codes-mcp-server/issues/4)). The `@fileoverview`, tool description, `node` field, and `axes` output description no longer promise partial-code axis expansion. Only the top-level Section axis (position 1) is browsable; positions 2–7 are context-dependent on the preceding axis path and aren't enumerable from a flat partial code, so a partial PCS `node` returns no axes plus a notice. The test fixture (`build-fixture-db.ts`) now seeds only position-1 Section axes, mirroring the shipped index.
- **`@types/node`** (dev) `^25.9.3` → `^26.0.0`.

## Fixed

- **`medcode_map_codes` resolvable-but-no-edge is a success, not an error** ([#5](https://github.com/cyanheads/medical-codes-mcp-server/issues/5)). A resolvable code with no edge in the requested direction — an ICD-10-PCS code (no prefix parent) or a top-level ICD-10-CM/HCPCS code — now returns `hits: []` with a `ctx.enrich.notice(...)`, matching `medcode_get_code` (`parent: null`) and `medcode_search_codes`/`medcode_browse_hierarchy` (empty + notice). `no_mapping` is now reserved for a `from` that does not resolve to any bundled code.
- **`medcode_check_code` names RxNorm as unbundled for out-of-scope numeric codes** ([#8](https://github.com/cyanheads/medical-codes-mcp-server/issues/8)). A bare integer matches only the RXCUI shape, but RxNorm is not bundled in this release, so a CPT / HCPCS Level I code such as `99213` previously came back as "No RXNORM code matches …" — implying a populated table was searched. It now reports RxNorm as not bundled and flags the likely CPT / HCPCS Level I origin (out of scope — this server bundles ICD-10-CM, ICD-10-PCS, HCPCS Level II), gated on `hasRxNorm()` so the message auto-corrects to a genuine per-concept not-found once RxNorm lands. The `medcode_browse_hierarchy` empty-result notice for `system: RXNORM` is aligned to the same "not bundled" phrasing.
