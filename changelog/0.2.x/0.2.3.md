# 0.2.3 — 2026-07-13

## Fixed

- **`medcode_browse_hierarchy`** reported `unknown_node` for a complete, existing ICD-10-PCS code. `browsePcs` now checks existence first — an existing complete code returns empty axes with an explanatory notice; `unknown_node` is reserved for a code genuinely absent from the index. ([#13](https://github.com/cyanheads/medical-codes-mcp-server/issues/13))
- **Whitespace-only required strings** cleared `.min(1)` and reached the service layer — worst case, `medcode_map_codes`'s `name_to_rxcui` direction built a `LIKE '%%'` that matched every drug name. A shared `nonBlankString` Zod helper now rejects whitespace-only values at the schema boundary on `map_codes.from`, `search_codes.query`, `check_code.code`, `get_code.codes[]`, and `browse_hierarchy.node`. ([#14](https://github.com/cyanheads/medical-codes-mcp-server/issues/14))
- **`medcode_search_codes`** missed single-token compound/inflected matches — a "neuropathy" query dropped billable `mononeuropathy`/`polyneuropathy` siblings because prefix-on-token FTS matching required a shared token stem. A substring-match tier now recovers them within the same ordered query, so pagination still reconstructs one set. ([#6](https://github.com/cyanheads/medical-codes-mcp-server/issues/6))
