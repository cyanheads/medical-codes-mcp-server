# 0.2.0 — 2026-06-21

RxNorm joins the bundled corpus, closing the one gap left from v1. The drug-crosswalk directions and a first-class offline NDC decode are live; the server stays 100% offline and keyless at runtime — RxNorm is acquired once at build time over the keyless RxNav REST API.

## Added

- **RxNorm bundled** — the current RxNorm normalized drug set (ingredients, brand names, clinical/branded drugs, packs) with their NDC and ingredient/brand edges, baked into `data/medical-codes.db`: 55,867 concepts, 77,199 edges, 248,093 NDC maps. `medcode_list_systems` reports the RxNorm row (`RxNorm (current normalized set)`; no effective dates — RxNav has no monthly release label). ([#9](https://github.com/cyanheads/medical-codes-mcp-server/issues/9))
- **`medcode_map_codes` drug directions are live** — `name_to_rxcui`, `ndc_to_rxcui`, `rxcui_to_ndc`, `rxcui_to_ingredients`, `rxcui_to_brands` resolve against the bundled RxNorm tables instead of raising `direction_unavailable`. NDC inputs are accepted hyphenated or as bare 10/11 digits. ([#9](https://github.com/cyanheads/medical-codes-mcp-server/issues/9))
- **First-class offline NDC decode in `medcode_get_code`** — an NDC (hyphenated `0777-3105-02`, or bare 10/11-digit) resolves directly to its RxNorm product via the bundled NDC↔RxNorm map, returned in `found` with a new optional `source: "NDC"` field on the result. A hyphenated NDC with no bundled match is a real miss (reported in `notFound` with the normalized key); a bare digit run that misses the map falls through to RXCUI detection. The offline counterpart to openfda-mcp-server's live `openfda_lookup_ndc`. ([#10](https://github.com/cyanheads/medical-codes-mcp-server/issues/10))
- **`scripts/ingest/fetch-rxnav.ts`** (new) — build-time fetcher that caches RxNorm concepts, NDCs, and ingredient/brand edges from the keyless RxNav REST API under `<from-dir>/rxnav/` (`concepts.json` + `products.jsonl`). Run once before `build-index.ts`; no network at runtime.

## Changed

- **RxNorm acquisition moved off the RRF files** — `parseRxNorm()` (RRF pipe-delimited parser) is replaced by `parseRxNav()`, which reads the JSON cache the fetcher writes. NLM gates the RxNorm RRF bulk downloads — including the prescribable subset — behind UMLS/UTS authentication, which an offline keyless package can't depend on; RxNav serves the public-domain normalized vocabulary (never the UMLS-licensed source vocabularies), preserving redistribution. `ingestRxNorm()` in `scripts/build-index.ts` now reads `<from-dir>/rxnav/` rather than RRF files. ([#9](https://github.com/cyanheads/medical-codes-mcp-server/issues/9))
- **`medcode_browse_hierarchy` on `RXNORM`** returns an empty result with a notice steering to `medcode_search_codes` / `medcode_get_code` / `medcode_map_codes` — RxNorm is a flat drug vocabulary with no prefix hierarchy to walk.
- **`direction_unavailable` is now a build-fallback, not the default** — drug directions raise it only when a build carries no RxNorm tables (e.g. a custom `MEDCODE_DB_PATH`); the shipped index bundles RxNorm. Tool descriptions, server `instructions`, `medcode_get_code` not-found messaging, and `README` updated to drop the "phase 2 / not bundled" caveats.
- **`scripts/build-fixture-db.ts`** — adds a small RxNorm drug graph (ingredients, a brand, two products with NDC and ingredient/brand edges) so the drug-direction and NDC-decode paths resolve against real fixture rows in tests.
